# ModelGate PRD

## 背景

企业内部通常会同时运行多个 Agent、AI 应用和研发工具。它们如果直接连接模型供应商，会带来密钥分散、费用不可归因、限流不可控、模型故障无法统一降级和接口适配重复建设等问题。

ModelGate 的目标是在业务应用和模型供应商之间提供统一基础设施层，让企业可以集中管理模型接入、虚拟 Key、限流、额度、路由、用量和账单。

## 用户角色

### 平台管理员

负责：

- 接入模型供应商
- CRUD 模拟 OpenAI/ChatGPT Provider、模拟部署和模拟单价
- 创建企业和团队
- 在必要时为任意团队代管成员加入和移出
- 审批团队负责人申请的逐模型周期权限与额度
- 设置全局限流
- 查看平台运行状态

### 团队管理员

负责：

- 添加既有平台用户、移出本团队成员，并在获批范围内按模型分配周期额度
- 向平台管理员申请团队模型权限和周期额度
- 在获批的团队范围内向成员发放或收回个人逐模型权益
- 查看团队用量和账单
- 设置预算告警

### 开发者

负责：

- 使用分配给自己的虚拟 Key 调用模型
- 查看项目调用记录
- 查询错误原因
- 查看剩余额度

## 用户问题

- 真实模型密钥散落在多个项目里，泄露后难以快速吊销。
- 多团队共用 Provider 额度，无法判断费用归属。
- 多人共用同一个 Key 时无法准确判断个人用量，审计和吊销也不清晰。
- Provider 限流之外，企业内部还需要团队、成员、Key、模型维度的限流。
- 请求前不知道输出 Token，直接扣费容易导致并发超扣。
- 模型服务不稳定时，业务应用需要统一重试、降级和可观测错误。
- 各 Provider 接口差异导致每个业务系统重复适配。

## MVP 范围

### 控制面

- 企业、团队基础管理，以及管理员主动向团队发放逐模型周期权益。
- 平台用户目录与团队成员关系的分层管理；成员移出后保留调用和账单归因。
- 团队负责人申请、管理员审批的团队逐模型周期权益。
- 团队负责人为成员分配个人逐模型周期权益；开发者在自己的 Key 页面生成独立虚拟 Key。
- Provider、真实模型名、加密凭据池与单价配置。
- 平台用户管理；用户可被分配为单个团队的负责人或开发成员。
- RPM、TPM、并发和 Token 额度策略配置。
- 调用日志、用量记录、逐模型剩余额度和团队/个人仪表盘查询。
- 统一账号密码登录与平台管理员、团队管理员、开发者三类角色授权。

### 数据面

- OpenAI 风格 `/v1/chat/completions` 接口。
- API Key 鉴权和模型访问权限校验。
- Caffeine、Redis、MySQL 三级读取链路。
- Redis 单 Lua 的 Key/团队 RPM、团队 TPM 与 Key/团队/逻辑模型并发控制。
- Redis 对团队和成员两层逐模型周期额度冻结、结算和释放。
- 普通响应和 SSE 流式响应代理。
- Mock Provider。
- OpenAI-Compatible Provider 转发与凭据轮询。
- 首包前的单次凭据故障切换。
- 事务 Outbox 投递的 `UsageCompletedEvent`，以及额度结算、用量、账单、预算告警和审计消费者。

### 异步计量

- 用量事件消费者。
- 账单消费者。
- 消费幂等记录。
- 每日用量汇总。

## 非目标

MVP 不包含：

- 完整管理控制台视觉系统。
- Claude、通义、DeepSeek 等多个真实 Provider 的完整适配。
- 多地域路由。
- 语义缓存和 Prompt 缓存。
- 内容安全审核。
- 租户隔离线程池。
- Provider 账单回查。
- 复杂质量评分和智能路由。

这些能力放入后续路线图。

## 成功标准

- 开发者可以使用虚拟 Key 调用真实模型名，而不接触真实 Provider 密钥。
- OpenAI-Compatible Provider 的多把凭据不会暴露给调用方；Mock Provider 仍可用于离线演练。
- 超出 RPM、并发或 Token 额度时，请求被稳定拒绝并返回可诊断错误。
- 流式响应能边生成边返回，不能等待完整模型响应后一次性输出。
- 成功、失败、取消请求都能形成可追溯的请求记录和用量事件。
- 重复消费同一个 Usage Event 不会产生重复账单。
- Redis 数据和 MySQL 账本的职责边界清晰，不能把 Redis 当最终账本。

## 验收场景

- 创建组织、团队、成员授权和虚拟 Key 后，可以使用该 Key 调用 `/v1/chat/completions`。
- 禁用虚拟 Key 后，后续请求被拒绝，缓存不会长期保留旧状态。
- 设置低 RPM 后，并发请求中超过限制的请求被拒绝。
- 设置成员逐模型周期额度后，并发请求不会超过团队或个人的同模型可用额度；负责人只能在团队同模型上限内继续分配。
- Mock Provider 模拟慢响应时，SSE 能持续输出并正确释放并发计数。
- Mock Provider 模拟 429 或 500 时，网关执行有限重试并记录失败原因。
- 重放同一 Usage Event 时，账单和额度流水保持幂等。
